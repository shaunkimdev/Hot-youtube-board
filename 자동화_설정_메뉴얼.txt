[0] 이 시스템이 하는 일
------------------------------------------------------------
매일 정해진 시각에 GitHub 클라우드에서 자동으로:
 1) 한국·일본 유튜브 인기영상 수집 (라이브/쇼츠/공식법인/해외 영상 제외)
 2) 개인 크리에이터 채널의 '롱폼' 영상만 9개 주제별 Top3 선정
 3) 라이징스타(구독 1만↓ 조회수 폭발 롱폼) 발굴
 4) Gemini API로 영상 요약 + /watch 타임라인 자동 생성
 5) 웹 대시보드(site/) + 엑셀 생성 → GitHub Pages로 자동 공개
 ※ 내 PC가 꺼져 있어도 GitHub 서버에서 실행됨.

[1] 최초 1회 설정  (★ 이것만 끝내면 이후 자동)
------------------------------------------------------------

(1) API 키 시크릿 등록
   - GOOGLE_API_KEY : ★완료됨 (2026-06-23 등록, YouTube Data API 전용)
   - GEMINI_API_KEY : ☐ 직접 등록 필요 (Gemini 요약용 — GOOGLE_API_KEY와 별도 키)
       방법 A) 웹: 저장소 → Settings → Secrets and variables → Actions
               → New repository secret
               Name = GEMINI_API_KEY,  Secret = AIza... (본인 키)
       방법 B) PowerShell(새 창)에서:
               gh secret set GEMINI_API_KEY
               → 실행 후 키 값을 붙여넣고 Enter

   (키 발급처)
     - YouTube Data API v3 키 : Google Cloud Console → API 및 서비스
                                → 'YouTube Data API v3' 사용 설정 → 사용자 인증 정보
     - Gemini 키 : https://aistudio.google.com/apikey (무료 발급, 별도 결제수단 불필요)

(2) GitHub Pages 켜기
   저장소 → Settings → Pages
   → Build and deployment → Source = "GitHub Actions" 선택

(3) 첫 실행(수동)으로 확인
   저장소 → Actions 탭 → 왼쪽 'daily-youtube-dashboard'
   → 오른쪽 'Run workflow' 버튼 클릭
   → 2~3분 뒤 초록 체크 → 위 배포 주소 접속해 대시보드 확인

[2] 자동 실행 일정
------------------------------------------------------------

[8] 문제 해결 (Troubleshooting)
------------------------------------------------------------
 ▶ 워크플로우 실패(빨간 X)
   - Actions 탭에서 해당 실행 클릭 → 로그 확인.
   - 가장 흔한 원인: 시크릿 미등록 → [1](1) 다시 확인.

 ▶ 'Search Queries per day' 쿼터 초과 (라이징스타 수집 실패)
   - YouTube 검색 API 일일 한도 초과. 태평양시간 자정에 리셋.
   - 해결: rising_search.py 의 검색 시드(SEED) 수를 줄이거나,
           Google Cloud 콘솔에서 쿼터 상향 신청(무료).

 ▶ 영상 다운로드 403/429 (프레임 분석 일부 실패)
   - GitHub 클라우드 IP를 유튜브가 막는 경우.
   - 해결: 브라우저 확장 'Get cookies.txt LOCALLY'로 youtube.com 쿠키를
           내보내 → 시크릿 YT_COOKIES 에 파일 내용 붙여넣기.
           (없어도 자막/텍스트 요약은 대부분 동작, 자동 폴백됨)
   - 또는 워크플로우 env의 DEEP_MAX 값을 5~8로 낮춰 다운로드 횟수 감소.

 ▶ Pages 주소가 안 열림
   - Settings→Pages에서 Source가 "GitHub Actions"인지 확인.
   - 첫 배포 후 1~2분 대기. Actions의 deploy-pages 단계 성공 확인.

 ▶ 예약 실행이 멈춤
   - GitHub는 60일간 저장소 활동이 없으면 예약(cron)을 자동 비활성화.
   - 본 워크플로우는 매일 결과를 커밋하므로 보통 유지됨.
   - 멈췄다면 Actions 탭에서 'Enable workflow' 한 번 클릭.
